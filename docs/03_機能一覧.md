## 3-2. パラメータ仕様

| 機能ID | パラメータ | 取得方法 | 型 | 必須 | 既定/補足 |
| ------ | ---------- | -------- | -- | ---- | --------- |
| FNC-06 | month      | `@RequestParam` | String | 任意 | 未指定/不正は当月 |
| FNC-07 | month      | `@RequestParam` | String | 任意 | 同上 |
| FNC-08 | type       | `@RequestParam` | TransactionType | 任意 | 既定 `EXPENSE` |
| FNC-09 | （フォーム） | `@Valid TransactionForm` | ― | ― | バリデーション発動 |
| FNC-10 | id / month | `@PathVariable` / `@RequestParam` | Long / String | id:必須 | 削除後に戻る月 |
| FNC-12 | type       | `@RequestParam` | TransactionType | 任意 | 既定 `EXPENSE` |
| FNC-14 | id         | `@PathVariable` | Long | 必須 | |
| FNC-15 | id / （フォーム） | `@PathVariable` / `@Valid CategoryForm` | Long | 必須 | URLのidをFormへ反映 |
| FNC-16 | id         | `@PathVariable` | Long | 必須 | |

## 3-3. ログイン中ユーザーの取得

要認証機能（FNC-06〜16）は、ログイン中ユーザーを **URLからではなく認証情報から** 取得する。

```text
@AuthenticationPrincipal UserDetails principal
        ↓ principal.getUsername() … email
UserService.findByEmail(email) … 本人の User Entity
```

詳細は [09_セキュリティ設計.md](./09_セキュリティ設計.md) を参照。
