# 📐 第12章 クラス別メソッド定義書

[← 目次に戻る](./README.md)

各クラスの**公開メソッド**について、シグネチャ・引数・戻り値・例外・処理概要を定義する。
フィールド定義は [05_クラス設計.md](./05_クラス設計.md)、処理フローは [06_処理設計.md](./06_処理設計.md)、
バリデーションは [07_バリデーション定義.md](./07_バリデーション定義.md) と対応する。

---

## 12-0. 凡例・共通事項

- **Lombok生成メソッドは列挙しない**：`@Getter`/`@Setter`/`@NoArgsConstructor`/`@AllArgsConstructor` による
  getter・setter・コンストラクタは自動生成のため、本書では個別行にしない（Entity/Form/DTO が該当）。
- **例外の意味**（詳細は [10_例外処理と共通部品設計.md](./10_例外処理と共通部品設計.md)）：

  | 例外 | 意味 | 最終的な結果 |
  | ---- | ---- | ------------ |
  | `ResourceNotFoundException` | 対象が存在しない／他人のもの | HTTP 404 |
  | `IllegalArgumentException` | 引数不正（重複・整合違反） | Controllerが捕捉→フォーム全体エラーで再描画 |
  | `IllegalStateException` | 状態不正（使用中で削除不可） | Controllerが捕捉→`errorMessage`フラッシュ |
  | `UsernameNotFoundException` | 認証時にユーザー不在 | Spring Securityが処理→`/login?error` |

- **本人特定パターン**（要認証Controller共通）：`userService.findByEmail(principal.getUsername())` で
  `@AuthenticationPrincipal` の email からログイン中 `User` を引き直す（URLの値は信用しない）。

## 12-1. Controller 層

### 12-1-1. `HomeController`

| メソッド（HTTP・URL） | 引数 | 戻り値 | 処理概要（例外） |
| --------------------- | ---- | ------ | ---------------- |
| `index()`<br/>GET `/` | ― | `String` = `"redirect:/dashboard"` | 常にダッシュボードへ内部リダイレクト。未ログインならSecurityが`/login`へ回す |

### 12-1-2. `LoginController`

| メソッド（HTTP・URL） | 引数 | 戻り値 | 処理概要（例外） |
| --------------------- | ---- | ------ | ---------------- |
| `showLoginForm(Model)`<br/>GET `/login` | `model` | `String` = `"auth/login"` | 空の`LoginForm`を`model`に入れて画面表示。**POST `/login`は書かない**（Securityが処理） |

### 12-1-3. `UserRegisterController`

| メソッド（HTTP・URL） | 引数 | 戻り値 | 処理概要（例外） |
| --------------------- | ---- | ------ | ---------------- |
| `showForm(Model)`<br/>GET `/register` | `model` | `String` = `"auth/register"` | 空の`UserRegisterForm`を入れて登録画面表示 |
| `register(...)`<br/>POST `/register` | `@Valid UserRegisterForm`, `BindingResult`, `Model`, `RedirectAttributes` | `String`（`"auth/register"` or `"redirect:/login"`） | 入力エラー→再描画。`userService.register`委譲。**`IllegalArgumentException`**（email重複）→`reject`して再描画。成功→フラッシュ`message`＋PRGで`/login`へ |

### 12-1-4. `DashboardController`

| メソッド（HTTP・URL） | 引数 | 戻り値 | 処理概要（例外） |
| --------------------- | ---- | ------ | ---------------- |
| `dashboard(...)`<br/>GET `/dashboard` | `@RequestParam month`（任意）, `@AuthenticationPrincipal UserDetails principal`, `Model` | `String` = `"dashboard"` | 本人特定→`parseMonthOrCurrent`→`userName`/`selectedMonth`/`availableMonths`/`summary`/`breakdown`/`trend`を`model`へ。描画はChart.js |

### 12-1-5. `TransactionController`

| メソッド（HTTP・URL） | 引数 | 戻り値 | 処理概要（例外） |
| --------------------- | ---- | ------ | ---------------- |
| `list(...)`<br/>GET `/transactions` | `@RequestParam month`（任意）, `principal`, `Model` | `String` = `"transaction/list"` | 本人特定→指定月の履歴一覧・月セレクタ用データを`model`へ |
| `showForm(...)`<br/>GET `/transactions/new` | `@RequestParam type`（既定`EXPENSE`）, `principal`, `Model` | `String` = `"transaction/form"` | 空`TransactionForm`に種類・今日の日付を初期設定。種類に応じたカテゴリー選択肢を`model`へ |
| `register(...)`<br/>POST `/transactions` | `@Valid TransactionForm`, `BindingResult`, `principal`, `Model`, `RedirectAttributes` | `String`（再描画 or `redirect:/transactions?month=取引月`） | 入力エラー→カテゴリー詰め直して再描画。`transactionService.register`委譲。**`IllegalArgumentException`**（種類不一致）→`reject`して再描画。成功→フラッシュ＋PRG |
| `delete(...)`<br/>POST `/transactions/{id}/delete` | `@PathVariable Long id`, `@RequestParam month`（任意）, `principal`, `RedirectAttributes` | `String` = `redirect:/transactions[?month=]` | 本人特定→`transactionService.delete`委譲→フラッシュ`message`。削除は必ずPOST |

### 12-1-6. `CategoryController`（`@RequestMapping("/categories")`）

| メソッド（HTTP・URL） | 引数 | 戻り値 | 処理概要（例外） |
| --------------------- | ---- | ------ | ---------------- |
| `list(...)`<br/>GET `/categories` | `principal`, `Model` | `String` = `"category/list"` | 支出/収入カテゴリーを別々に取得して`model`へ |
| `showRegisterForm(...)`<br/>GET `/categories/new` | `@RequestParam type`（既定`EXPENSE`）, `Model` | `String` = `"category/form"` | 空`CategoryForm`に種類・初期色を設定。`isEdit=false` |
| `register(...)`<br/>POST `/categories` | `@Valid CategoryForm`, `BindingResult`, `principal`, `Model`, `RedirectAttributes` | `String`（再描画 or `redirect:/categories`） | 入力エラー→再描画（`isEdit=false`）。`register`委譲。**`IllegalArgumentException`**（同名重複）→`reject`。成功→フラッシュ＋PRG |
| `showEditForm(...)`<br/>GET `/categories/{id}/edit` | `@PathVariable Long id`, `principal`, `Model` | `String` = `"category/form"` | `toEditForm`でDB→Form詰め替え。`isEdit=true`。**`ResourceNotFoundException`**（他人/不在） |
| `update(...)`<br/>POST `/categories/{id}/edit` | `@PathVariable Long id`, `@Valid CategoryForm`, `BindingResult`, `principal`, `Model`, `RedirectAttributes` | `String`（再描画 or `redirect:/categories`） | URLのidをFormへ反映→入力エラー→再描画（`isEdit=true`）。`update`委譲。**`IllegalArgumentException`**→`reject`。成功→PRG |
| `delete(...)`<br/>POST `/categories/{id}/delete` | `@PathVariable Long id`, `principal`, `RedirectAttributes` | `String` = `redirect:/categories` | `delete`委譲。**`IllegalStateException`**（使用中）を捕捉→`errorMessage`フラッシュ。成功→`message` |

## 12-2. Service 層

### 12-2-1. `UserService`（`implements UserDetailsService`）

| メソッド | 引数 | 戻り値 | 処理概要（例外） |
| -------- | ---- | ------ | ---------------- |
| `loadUserByUsername(String email)` | email | `UserDetails` | Security認証の窓口。emailで`User`を引き、ハッシュ済パスワード＋`ROLE_USER`で`UserDetails`を組み立てる。**`UsernameNotFoundException`**（不在） |
| `findByEmail(String email)` | email | `User` | emailから本人Entityを取得。**`ResourceNotFoundException`**（不在） |
| `register(UserRegisterForm form)`<br/>`@Transactional` | form | `User` | email重複チェック→`User`生成→**パスワードをBCryptハッシュ化**→保存→初期カテゴリー12件作成。**`IllegalArgumentException`**（email重複） |
| `createDefaultCategories(User)`<br/>*private* | user | `void` | 支出8件＋収入4件の初期カテゴリーを作成（[04_DB設計 4-4-1](./04_DB設計.md)） |
| `createCategory(User, TransactionType, String, String)`<br/>*private* | user, type, label, color | `void` | カテゴリー1件を生成・保存する補助メソッド |

### 12-2-2. `CategoryService`

| メソッド | 引数 | 戻り値 | 処理概要（例外） |
| -------- | ---- | ------ | ---------------- |
| `findAllByUser(User user)` | user | `List<Category>` | 指定ユーザーの全カテゴリー（ID昇順） |
| `findByUserAndType(User, TransactionType)` | user, type | `List<Category>` | 指定タイプ（支出/収入）のカテゴリー（ID昇順）。記録画面の選択肢用 |
| `findOwnedById(User, Long id)` | user, id | `Category` | ID取得＋**持ち主チェック**。他人/不在は同一扱い。**`ResourceNotFoundException`** |
| `toEditForm(User, Long id)` | user, id | `CategoryForm` | 自分のCategoryを取得しForm（id/type/label/color）に詰め替え。**`ResourceNotFoundException`** |
| `register(User, CategoryForm)`<br/>`@Transactional` | user, form | `Category` | 同一user＋type内の同名重複チェック→生成（user＝ログイン固定）→保存。**`IllegalArgumentException`**（重複） |
| `update(User, CategoryForm)`<br/>`@Transactional` | user, form | `Category` | 持ち主取得→名前変更時のみ重複チェック→label/color更新（**typeは変更しない**）。**`ResourceNotFoundException`** / **`IllegalArgumentException`** |
| `delete(User, Long id)`<br/>`@Transactional` | user, id | `void` | 持ち主取得→`countByCategory`で使用件数確認→0件のみ削除。**`ResourceNotFoundException`** / **`IllegalStateException`**（使用中） |

### 12-2-3. `TransactionService`

| メソッド | 引数 | 戻り値 | 処理概要（例外） |
| -------- | ---- | ------ | ---------------- |
| `findByMonth(User, YearMonth month)` | user, month | `List<Transaction>` | 月初〜月末（Between両端含む）の記録を新しい順で取得 |
| `summarize(User, YearMonth)` | user, month | `MonthlySummary` | 月の記録をtypeで振り分け合計。`balance = income − expense` |
| `expenseBreakdown(User, YearMonth)` | user, month | `List<CategorySlice>` | 支出のみカテゴリー別合計→割合算出（分母0は0）→金額降順ソート |
| `recentTrend(User, YearMonth current)` | user, current | `List<MonthlyTrendPoint>` | 当月含む直近6ヶ月を古い順に`summarize`し `label="N月"` でリスト化 |
| `availableMonths(YearMonth current)` | current | `List<String>` | 直近12ヶ月を`"yyyy-MM"`で新しい順に返す（月セレクタ用） |
| `parseMonthOrCurrent(String monthStr)` | monthStr | `YearMonth` | `"yyyy-MM"`を変換。null/空/不正は**当月にフォールバック**（例外で落とさない） |
| `register(User, TransactionForm)`<br/>`@Transactional` | user, form | `Transaction` | `categoryId`→`Category`変換（`findOwnedById`で持ち主チェック）→種類一致検証→生成（user＝ログイン固定）→保存。**`ResourceNotFoundException`** / **`IllegalArgumentException`**（種類不一致） |
| `delete(User, Long id)`<br/>`@Transactional` | user, id | `void` | 記録取得→自分の記録か確認→削除。**`ResourceNotFoundException`**（他人/不在） |

## 12-3. Repository 層

> いずれも `JpaRepository<Entity, Long>` を継承し、`save` / `findById` / `findAll` / `delete` / `deleteById` 等の
> 基本CRUDを自動で備える。以下は**メソッド名規約で自動生成される追加メソッド**のみ記載。

### 12-3-1. `UserRepository`

| メソッド | 引数 | 戻り値 | 自動生成SQL（概要） |
| -------- | ---- | ------ | ------------------- |
| `findByEmail(String email)` | email | `Optional<User>` | `WHERE email = ?`。不在の可能性があるため`Optional` |
| `existsByEmail(String email)` | email | `boolean` | `count(*) > 0 WHERE email = ?`。登録時の重複チェック |

### 12-3-2. `CategoryRepository`

| メソッド | 引数 | 戻り値 | 自動生成SQL（概要） |
| -------- | ---- | ------ | ------------------- |
| `findByUserOrderByIdAsc(User user)` | user | `List<Category>` | `WHERE user_id = ? ORDER BY id ASC` |
| `findByUserAndTypeOrderByIdAsc(User, TransactionType)` | user, type | `List<Category>` | `WHERE user_id = ? AND type = ? ORDER BY id ASC` |
| `existsByUserAndTypeAndLabel(User, TransactionType, String)` | user, type, label | `boolean` | `count(*) > 0 WHERE user_id=? AND type=? AND label=?` |

### 12-3-3. `TransactionRepository`

| メソッド | 引数 | 戻り値 | 自動生成SQL（概要） |
| -------- | ---- | ------ | ------------------- |
| `findByUserAndTransactionDateBetweenOrderByTransactionDateDescIdDesc(User, LocalDate, LocalDate)` | user, start, end | `List<Transaction>` | `WHERE user_id=? AND transaction_date BETWEEN ? AND ? ORDER BY transaction_date DESC, id DESC`（月区切りの主要クエリ） |
| `countByCategory(Category category)` | category | `long` | `count(*) WHERE category_id = ?`。カテゴリー削除前チェック用 |

## 12-4. Entity 層

> フィールドの getter/setter は Lombok（`@Getter`/`@Setter`）生成のため省略。以下は独自メソッドのみ。

### 12-4-1. `TransactionType`（enum）

| メソッド | 引数 | 戻り値 | 処理概要 |
| -------- | ---- | ------ | -------- |
| `getLabel()` | ― | `String` | 定数が持つ画面表示ラベル（`EXPENSE`→"支出" / `INCOME`→"収入"）を返す |

### 12-4-2. `User` / `Category` / `Transaction`（ライフサイクルコールバック）

3クラスとも同一シグネチャの自動コールバックを持つ（`protected void`）。

| メソッド | トリガー | 処理概要 |
| -------- | -------- | -------- |
| `onCreate()` | `@PrePersist`（INSERT直前） | `createdAt`・`updatedAt` に現在時刻を同値でセット |
| `onUpdate()` | `@PreUpdate`（UPDATE直前） | `updatedAt` のみ現在時刻で上書き（`createdAt`は不変） |

## 12-5. Form / DTO

いずれも Lombok によるデータクラスで、独自メソッドを持たない（メソッド定義は getter/setter/コンストラクタのみ＝自動生成）。

| クラス | Lombok | メソッド | 補足 |
| ------ | ------ | -------- | ---- |
| `LoginForm` / `UserRegisterForm` / `CategoryForm` / `TransactionForm` | `@Getter @Setter @NoArgsConstructor` | 各フィールドの getter/setter | 検証は各フィールドのアノテーション（[07](./07_バリデーション定義.md)）。フィールド定義は [05-2](./05_クラス設計.md) |
| `MonthlySummary` / `CategorySlice` / `MonthlyTrendPoint` | `@Getter @AllArgsConstructor` | getter＋全項目コンストラクタ | 不変の運び箱（DB非結合）。フィールド定義は [05-3](./05_クラス設計.md) |

## 12-6. config / exception

### 12-6-1. `SecurityConfig`

| メソッド | 引数 | 戻り値 | 処理概要（例外） |
| -------- | ---- | ------ | ---------------- |
| `passwordEncoder()`<br/>`@Bean` | ― | `PasswordEncoder` | `BCryptPasswordEncoder`を返す（salt自動付与・一方向）。`UserService`が注入して使う |
| `filterChain(HttpSecurity http)`<br/>`@Bean` | http | `SecurityFilterChain` | 認可（`permitAll`→`authenticated`の順）・フォーム認証（`usernameParameter("email")`・成功時`/dashboard`）・ログアウト（`/login?logout`）を設定。**`throws Exception`** |

### 12-6-2. `ResourceNotFoundException`（`extends RuntimeException`）

| メソッド | 引数 | 戻り値 | 処理概要 |
| -------- | ---- | ------ | -------- |
| `ResourceNotFoundException(String message)` | message | （コンストラクタ） | メッセージ付きでスロー。クラスに`@ResponseStatus(NOT_FOUND)`が付き、未捕捉ならHTTP 404になる |
